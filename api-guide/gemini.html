Google Gemini API

Medium ~10 min real time 8 steps Verified: Feb 18, 2026

Get a Google Gemini API key, understand the models, send your first request. The hard part isn't the API — it's figuring out which Google dashboard you're supposed to be in.

> TL;DR

Go to aistudio.google.com → Sign in with Google → Click "Get API Key" → Create API key → Copy it → Use it with generativelanguage.googleapis.com. Free tier = 15 RPM for Flash. That's it. Don't go to Google Cloud Console unless you need billing.

> What You Need Before Starting

  • A Google account (Gmail works)
  • No credit card needed for free tier
Step 1

Go to Google AI Studio

Navigate to aistudio.google.com. This is NOT the Google Cloud Console. They are two completely different things and this is where 90% of people get confused.

AI Studio is Google's playground for Gemini. It's where you prototype, test prompts, and — critically — generate API keys. Google Cloud Console is for production infrastructure, billing, IAM, and existential dread.

// GOTCHA — AI STUDIO vs CLOUD CONSOLE

Two completely different dashboards. AI Studio (aistudio.google.com) is for quick prototyping and API keys. Cloud Console (console.cloud.google.com) is for production, billing, and IAM. If you're staring at a page full of "Enable APIs" and "Service Accounts," you're in the wrong place. Go back to AI Studio.

Step 2

Sign in with Your Google Account

If you have multiple Google accounts, pick carefully. The API key gets tied to whichever account you sign in with.

You can't transfer an API key to a different account later. If you want to switch, you'll need to create a brand new key under the other account.

// PRO TIP

If you're building something for a team or a project, use a dedicated Google account (like myproject.api@gmail.com) rather than your personal one. Makes it easier to hand off later and keeps your personal account's usage clean.

Step 3

Click "Get API Key" in the Left Sidebar

Once you're in AI Studio, look at the left sidebar. You'll see "Get API Key" — click it.

It's not in Settings. It's not in your profile. It's not behind a gear icon. It's right there in the sidebar, but somehow everyone misses it the first time because they're looking for something more complicated.

Step 4

Click "Create API Key"

It will ask which Google Cloud project to associate the key with. If you don't have a project yet, no problem — just click "Create API key in new project" and it creates one automatically.

If you already have Google Cloud projects, you can pick an existing one. Doesn't matter much for the free tier.

// GOTCHA

The key format starts with AIzaSy and is 39 characters long. If your key doesn't start with AIzaSy, you might have accidentally generated a Cloud Console service account key (a JSON file). That's a completely different authentication mechanism. Go back to AI Studio and get the right one.

Step 5

Copy the Key

Your API key will be displayed on screen. Format: AIzaSy... (39 characters). Copy it NOW.

You can find it again later in AI Studio under "Get API Key," but it's buried and annoying to navigate back to. Save yourself the trouble — copy it to your .env file immediately.

GEMINI_API_KEY=AIzaSyD...your-key-here
// PRO TIP

Never hardcode this key in source files. Store it in an environment variable or a .env file that's in your .gitignore. If the key leaks, go back to AI Studio, delete the key, and create a new one. Takes 30 seconds.

Step 6

Test It with curl

Send your first request to make sure the key works:

curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"contents":[{"parts":[{"text":"Hello"}]}]}'

You should get back a JSON response with the model's reply. If you get {"error":{"code":400}}, check your key. If you get 403, your key might need billing enabled (see Step 8).

{
  "candidates": [{
    "content": {
      "parts": [{"text": "Hello! How can I help you today?"}],
      "role": "model"
    }
  }]
}
// GOTCHA — v1 vs v1beta

The API endpoint uses /v1beta/, not /v1/. The v1beta version has more features and more models available. If you use /v1/, some models and features won't work. Yes, it's been "beta" for a long time. Welcome to Google.

Step 7

Understand the Models

This is where it gets messy. Google renames, removes, and adds models constantly. As of February 2026:

Main models:

gemini-2.5-flash-image — Latest Flash model with image generation. Fast, cheap, capable.

gemini-2.0-flash — Fast general-purpose model. Good for most tasks.

gemini-2.0-flash-exp-image-generation — Experimental image generation fallback.

gemini-1.5-pro — Best quality for complex reasoning. Slower, lower rate limits on free tier.

// GOTCHA — MODEL NAMES CHANGE

Google removes models without much warning. gemini-2.0-flash-preview-image-generation was REMOVED and now returns 404. If your code suddenly starts failing, check the current model list at aistudio.google.com first. The model you were using yesterday might not exist today.

// PRO TIP

For image generation, use gemini-2.5-flash-image as your primary model with gemini-2.0-flash-exp-image-generation as a fallback. Always have a fallback — Google will deprecate your primary model eventually.

Step 8

Know the Rate Limits

Free tier rate limits are per-minute, not per-day:

Flash models: 15 requests per minute (RPM). Fine for testing and personal projects. Terrible for anything with real users.

Pro models: 2 requests per minute (RPM). Barely enough to test with.

If you need more, you have to enable billing in Google Cloud Console (the other dashboard). Link a billing account to your project, and the limits increase dramatically.

// GOTCHA — 403 ERRORS

Some models require billing enabled even though there's technically a free tier. If you get a 403 "permission denied" error, it usually means you need to go to console.cloud.google.com, find your project, and link a billing account. You might not get charged anything, but Google needs the billing account on file.

// GOTCHA — PER MINUTE, NOT PER DAY

15 RPM sounds generous until you realize it's per minute. If your app sends a burst of 20 requests, 5 will fail. Implement rate limiting and retry logic on your end. A simple delay queue with 4-second spacing keeps you safely under 15 RPM.

> What You End Up With

API Key AIzaSy... (39 chars)
Base URL generativelanguage.googleapis.com
Env Variable GEMINI_API_KEY
Rate Limits (Free) 15 RPM (Flash) / 2 RPM (Pro)
Cost $0 — free tier
// REAL TALK

The Gemini API itself is straightforward once you have the key. The real complexity is Google's ecosystem: AI Studio vs Cloud Console, model names that change every month, rate limits that differ by model and tier, and the v1/v1beta split. Bookmark aistudio.google.com and check the model list before debugging anything — half the time, the model you're calling just got renamed or removed. The API is good. The naming and organization around it is chaos.
