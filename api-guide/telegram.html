Telegram Bot API

Easy ~5 min real time 6 steps Verified: Feb 19, 2026

Create a Telegram bot, get the token, send your first message. This is the simplest API to start with — BotFather makes it almost painless. Almost.

> TL;DR

Talk to @BotFather on Telegram → /newbot → pick a name → get a token like 1234567890:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw → use it in HTTP requests to api.telegram.org/bot{TOKEN}/. That's it. The rest is details.

> What You Need Before Starting

  • A Telegram account (phone number required for signup)
  • The Telegram app installed (desktop or mobile, either works)
  • A way to make HTTP requests (curl, Node.js, Python, Postman — anything)
Step 1

Find BotFather

Open Telegram and search for @BotFather. It's the official Telegram bot for creating bots — very meta.

Make sure you're talking to the real one: it has a blue checkmark and the username is exactly @BotFather (capital B, capital F). There are fake ones.

// GOTCHA

If you search "botfather" (lowercase), you might see knockoff bots. The real one is always the first result with the verified badge. If it doesn't have the badge, it's not the real one.

Step 2

Create Your Bot

Send /newbot to BotFather. It will ask you two things:

1. Display name: This is the human-readable name users see. Can be anything, can include spaces and emoji. Example: Kingston Orchestrator

2. Username: Must end in bot. Must be unique across all of Telegram. No spaces. Example: kingston_bastilon_bot

// GOTCHA

The username must end in "bot" (case-insensitive). So my_api_bot works, MyAPIBot works, but my_api will be rejected. Also, if the username is taken, BotFather just says "Sorry, this username is already taken." — no suggestions. You have to guess a new one.

Step 3

Get Your Token

After you provide a valid username, BotFather immediately gives you a token. It looks like this:

8220356435:AAGQ_0ScANDqoSS4Lhv3DhN_cl9CQ_WfStE

The format is {bot_id}:{random_string}. The part before the colon is your bot's numeric ID.

Copy this token immediately. You can retrieve it later with /token in BotFather, but save yourself the trouble.

// PRO TIP

Store this in an environment variable, never hardcode it. If your token leaks (pushed to GitHub, shared in a screenshot), anyone can control your bot. If it happens: go back to BotFather and use /revoke to get a new token immediately.

Step 4

Find Your Chat ID

To send a message, you need the recipient's chat_id. The easiest way:

1. Open your bot in Telegram (search for its @username) and send it any message, like "hello".

2. Call the getUpdates endpoint:

curl https://api.telegram.org/bot{YOUR_TOKEN}/getUpdates

3. In the JSON response, look for message.chat.id. That number is your chat ID.

{
  "ok": true,
  "result": [{
    "message": {
      "chat": {
        "id": 8189338836,    // <-- THIS IS YOUR CHAT ID
        "type": "private"
      },
      "text": "hello"
    }
  }]
}
// GOTCHA

If getUpdates returns an empty array {"ok":true,"result":[]}, it means either: (1) you haven't sent any message to the bot yet, or (2) you're using webhooks, which consume the updates. Send a message to the bot first, then try again.

Step 5

Send Your First Message

Now you have the token and the chat ID. Send a message:

curl -X POST "https://api.telegram.org/bot{TOKEN}/sendMessage" \
  -H "Content-Type: application/json" \
  -d '{"chat_id": YOUR_CHAT_ID, "text": "Hello from the API!"}'

If you get {"ok":true} back, you're done. Your bot just sent a message.

// GOTCHA — UTF-8 ENCODING

If you're sending text with accented characters (French, Spanish, etc.) via curl, it might break. Curl doesn't always handle UTF-8 well depending on your terminal. Use a proper HTTP library (Node.js https module, Python requests, etc.) instead of curl for production. This one bit us hard with French text.

Step 6

Send a Photo (Bonus)

Sending photos uses multipart/form-data, which is slightly more complex:

curl -X POST "https://api.telegram.org/bot{TOKEN}/sendPhoto" \
  -F "chat_id=YOUR_CHAT_ID" \
  -F "photo=@./my_image.png" \
  -F "caption=Check out this image!"

Or send by URL (no upload needed):

curl -X POST "https://api.telegram.org/bot{TOKEN}/sendPhoto" \
  -H "Content-Type: application/json" \
  -d '{"chat_id": YOUR_CHAT_ID, "photo": "https://example.com/image.png"}'
// PRO TIP

Telegram caches images by URL. If you update the image at the same URL, Telegram will keep showing the old one. Add a query parameter like ?v=2 to bust the cache.

Env Variables Needed TELEGRAM_BOT_TOKEN, CHAT_ID
Rate Limits ~30 msg/sec to different chats
Cost $0 — forever free
// REAL TALK

Telegram's Bot API is genuinely one of the best-documented, simplest APIs out there. If every API was this easy, this website wouldn't exist. BotFather is the gold standard that every other API provider should study. The only issues are: (1) UTF-8 in curl, (2) empty getUpdates when webhooks are set, and (3) the username uniqueness guessing game. Otherwise? Chef's kiss.
