Alpaca Trading API

Medium ~15 min real time 10 steps Verified: Feb 18, 2026

Set up Alpaca for paper trading or live stock/crypto trading. Get your API keys, test with $100K in fake money, and learn the one-character difference between paper and real money before it costs you.

> TL;DR

Sign up at alpaca.markets → Verify email → Go to Paper Trading dashboard → Click "View" under API Keys → Copy key and secret → Use paper URL paper-api.alpaca.markets for testing. DO NOT use api.alpaca.markets until you know what you're doing.

> What You Need Before Starting

  • An email address
  • No SSN or ID needed for paper trading — just sign up and go
  • Real (live) trading requires: SSN, proof of address, takes 1–3 business days to approve
  • A way to make HTTP requests (curl, Node.js, Python — anything)
Step 1

Sign Up at alpaca.markets

Go to alpaca.markets and click "Sign Up". Use your real email — they verify it. You can sign up with Google or email/password.

Don't overthink it. Paper trading accounts are free, instant, and come with $100,000 in fake money.

Step 2

Verify Your Email

Click the verification link in the email Alpaca sends you. Check your spam folder — it sometimes lands there, especially with Gmail.

If you signed up with Google OAuth, this step is automatic.

Step 3

Choose Paper Trading

After login, you'll land on the Alpaca dashboard. Paper trading is enabled by default. You should see a toggle or dropdown at the top that says "Paper Trading" or "Live Trading".

Make sure it says Paper Trading. It comes with $100,000 in fake money. Same API, zero risk.

// GOTCHA — ACCOUNT STATUS

Even paper accounts have a "status" field. It should be ACTIVE. If it says ONBOARDING, wait a few minutes and refresh. It usually resolves itself. If it doesn't after 10 minutes, log out and log back in.

Step 4

Find the API Keys Section

Dashboard → left sidebar → look for "API Keys" or go to the Overview page and look for the API key section.

It's NOT in Settings. This trips people up. The API keys live on the main dashboard area, usually in a card on the Overview page or under a dedicated "API Keys" tab in the sidebar.

Step 5

Generate API Keys

Click "Regenerate" or "View" to see your API key and secret.

CRITICAL: The SECRET is only shown ONCE. The moment you close that dialog, it's gone. You'll need to regenerate if you lose it.

// GOTCHA — SECRET SHOWN ONCE

We cannot stress this enough. When the modal pops up with your key and secret, copy both immediately. Paste them somewhere safe before clicking anything else. If you close that dialog without copying the secret, you have to regenerate everything and update it everywhere you've used the old key.

Step 6

Copy BOTH Keys

You need two things for every API call:

1. APCA-API-KEY-ID — This is your API key. Looks like PKxxxxxxxxxxxxxxxx

2. APCA-API-SECRET-KEY — This is your secret. A long alphanumeric string.

Both are sent as HTTP headers with every request. Miss one, get a 401.

Step 7

Set Up Environment Variables

Store your keys in environment variables. Never hardcode them.

ALPACA_API_KEY=PKxxxxxxxxxxxxxxxxxxxx
ALPACA_SECRET_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
ALPACA_BASE_URL=https://paper-api.alpaca.markets

In Node.js with dotenv, in Python with python-dotenv, in your .env file — however your stack does it.

// PRO TIP

Add .env to your .gitignore before you do anything else. One accidental push to GitHub and your keys are public. Alpaca does monitor for leaked keys and will email you, but don't rely on it.

Step 8

Test with curl

Hit the account endpoint to verify everything works:

curl -H "APCA-API-KEY-ID: YOUR_KEY" \
     -H "APCA-API-SECRET-KEY: YOUR_SECRET" \
     https://paper-api.alpaca.markets/v2/account

You should get a JSON response with your account details:

{
  "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "status": "ACTIVE",
  "currency": "USD",
  "buying_power": "400000",
  "portfolio_value": "100000",
  "pattern_day_trader": false,
  ...
}

If you see "status": "ACTIVE" and "portfolio_value": "100000", you're golden.

// GOTCHA — INVALID API KEY

If you get "message": "forbidden." or a 403, double-check: (1) You're using the right base URL for your account type (paper vs live). (2) You didn't mix up key and secret. (3) Your keys aren't from a different account. (4) You didn't accidentally regenerate keys and invalidate the old ones.

Step 9

Understand Paper vs Live

This is the most important step. Read it twice.

Paper: paper-api.alpaca.markets

Live: api.alpaca.markets

ONE PREFIX in your base URL is the difference between fake money and real money. That's it. Same API, same endpoints, same headers. Just the URL changes.

// GOTCHA — PAPER vs LIVE URL

The ONLY difference is the paper- prefix. One typo, one wrong environment variable, one copy-paste from Stack Overflow with the live URL — and you're trading with real money. Triple-check your ALPACA_BASE_URL. Then check it again.

// PRO TIP

Use the Alpaca SDK instead of raw HTTP calls: npm install @alpacahq/alpaca-trade-api. The SDK lets you set paper mode with a boolean flag (paper: true) instead of swapping URLs manually. Much harder to mess up.

Step 10

Check Market Hours

Stocks aren't 24/7. Know your windows:

Regular hours: Mon–Fri 9:30 AM – 4:00 PM ET

Extended hours: 4:00 AM – 8:00 PM ET

Outside these hours, stock orders get queued, not rejected. They sit there silently until the market opens.

// GOTCHA — QUEUED ORDERS

Orders placed outside market hours don't get rejected — they get queued. You might forget about them, and they execute Monday at 9:30 AM. Surprise! Always check your open orders before market open if you were testing over the weekend.

// GOTCHA — CRYPTO IS 24/7

Unlike stocks, crypto trading on Alpaca is 24/7. Different rules, different behavior. Crypto orders execute immediately, any time, any day. Don't assume stock market hours apply to crypto.

// GOTCHA — RATE LIMITS

Paper trading has a rate limit of 200 requests per minute. Hit that ceiling and you'll get 429 errors. If you're polling every second, you'll burn through that fast. Use WebSocket streams for real-time data instead of polling.

// GOTCHA — API VERSION

The v2 API is what you want. Don't use v1 endpoints — they're deprecated. Every URL should have /v2/ in it.

// PRO TIP

Start with paper trading, always. $100K fake money, same API, zero risk. The WebSocket stream for real-time market data is at wss://stream.data.alpaca.markets/v2/iex — way better than polling the REST API.

> What You End Up With

API Key ID PKxxxxxxxxxxxxxxxx
Secret Key xxxxxxxxxxxxx (shown once!)
Paper URL paper-api.alpaca.markets/v2/
Live URL api.alpaca.markets/v2/
Env Variables Needed ALPACA_API_KEY, ALPACA_SECRET_KEY, ALPACA_BASE_URL
Rate Limits 200 req/min (paper)
Cost $0 paper — $0 live (commission-free)
// REAL TALK

Alpaca is the best brokerage API for developers, hands down. Commission-free, paper trading with the same API as live, good docs, and a Node.js SDK that actually works. The only real danger is the paper-vs-live URL thing — it's terrifyingly easy to switch between fake money and real money. Respect that. Start with paper, get comfortable, and triple-check your base URL before going live. Your wallet will thank you.
